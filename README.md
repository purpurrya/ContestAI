# ContestAI — Платформа для соревнований ИИ-ботов

Проект выполнен в рамках курсовой работы по дисциплине «Веб-разработка» (ДВФУ, 2025).

## О проекте

ContestAI — это учебный прототип платформы для проведения интерактивных соревнований между алгоритмами искусственного интеллекта. Система позволяет участникам разрабатывать ботов, соревноваться в пошаговых играх, управляет состоянием матчей, проверяет ходы по правилам, сохраняет историю и предоставляет визуализацию партий.

Ключевые архитектурные решения: stateful-подход (боты могут хранить историю и адаптировать стратегию), разделение ответственности (игровая логика вынесена в отдельную библиотеку), интерактивный формат матча (последовательный обмен состояниями и ходами).

## Реализованные игры

- Упрощённый покер: однокарточный покер с фазами ставок (BET, CALL, CHECK, FOLD)
- Гексагональные шашки: доска в виде графа, три игрока, поддержка дамок и взятий

## Архитектура

Система состоит из следующих основных компонентов:

- Веб-приложение на Django: регистрация, аутентификация, профили пользователей
- Библиотека game_engine: независимая реализация правил игр и валидации ходов
- Сетевой прототип на Flask и Redis: обмен ходами между сервером и клиентами ботов
- Клиент бота (player_client.py): пример реализации участника
- Визуализатор шашек на HTML/Canvas: пошаговое воспроизведение партий

## Быстрый старт

1. Клонирование репозитория:

git clone https://github.com/purpurrya/ContestAI.git
cd ContestAI

2. Установка зависимостей:

pip install -r requirements.txt

3. Запуск Django-сервера (основной веб-интерфейс):

cd django_web
python manage.py migrate
python manage.py runserver

4. Запуск игрового сервера (требуется запущенный Redis):

python scripts/game_server.py

5. Запуск примера клиента бота (в другом терминале):

python scripts/player_client.py

## Тестирование

Запуск всех тестов:

pytest tests/

Тесты покрывают логику покера (ходы, фазы, ошибки), правила гексагональных шашек и сетевой сценарий взаимодействия клиента с сервером.
